# ホーム画面ワイヤーフレーム 2案（2026-07-18、v4.1 §12-3）

> 390pxスマホ基準。モックHTML＋スクショはチャットで提出済み（実装はしていない）。共通原則: ダッシュボード化しない・1画面目で次に押すボタンが分かる・はちゃんは伴走者。

## 案A: 学習導線重視

**思想**: 「今日やるべき1つ」を最速で提示する。はちゃんは現在のPhase1と同じ小ささ（44px）のまま。

### 1画面目（上から）
1. ヘッダー（既存1行52px・🔥ストリーク）
2. はちゃん44px＋一言（既存コンポーネント流用）
3. **「⚡今日のおすすめ」カード＝最優先CTA**: Daily未完了→「Daily Dashをやる（60秒）」／完了→「苦手N語を復習する」
4. 第2導線: 「📚 Studyを続ける（苦手3語から）」行ボタン
5. 成長チップ2つ: 今日思い出せた🧠N語／あと約N問でLv.X

### スクロール後
Challenge・Battleタイル → レベルバー → 学習データリンク →（カテゴリ・ミックス等は現状のまま下部）

### 特性
- 最優先CTA: 「今日のおすすめ」1ボタン（状態で中身が変わる）
- Daily: 未完了時は最上位。完了後はおすすめ枠がStudyに切替
- Study: 常に第2導線として1画面目に残る
- Challenge/Battle: fold下（学習を主、遊びを従とする配置）
- 情報量: 1画面目に7要素・テキスト中心
- **実装コスト: 小**。既存カード（はちゃん行・レベルバー・モードタイル）の再配置＋「おすすめカード」1個の新規。CSSは既存トーン踏襲。1〜2日

## 案B: はちゃん・ゲーム世界観重視

**思想**: ホーム＝はちゃんのいる場所。開いた瞬間に「世界」があり、CTAは1つだけ。将来の時間帯背景・季節装飾・衣装（v4.1 §9）の土台になる。

### 1画面目（上から）
1. ヘッダー（既存、「プレイ」→「ホーム」に改名）
2. **シーン領域**: 夜空グラデ背景（時間帯で変化する想定）＋はちゃん96px＋長めの一言（状態反映:「昨日の3語、今日も思い出せるか確かめよっか」）＋🔥連続・今日の思い出し数
3. コピー「今日も、はちゃんと少しだけ。」
4. **単一CTA「▶ 今日のぶんをはじめる」**（中身は自動: Daily未完了→Daily、完了→Study復習）＋次に待つ内容のヒント1行

### スクロール後
モード4タイル（Study/Challenge/Battle/記録）→ レベルバー

### 特性
- 最優先CTA: 完全に1つ。モード選択の認知負荷をゼロにする
- はちゃん: 96px＋シーン背景で存在感が主役級（※伴走者原則との緊張あり — セリフは応援のみで担保）
- Daily/Study: CTA内に隠蔽（「何をやるか」をアプリが決める）
- Challenge/Battle: fold下タイル
- 情報量: 1画面目に5要素・雰囲気重視
- **実装コスト: 中**。シーン領域（背景・レイアウト・状態別セリフ拡充）が新規。時間帯背景まで含めると+α。3〜5日

## 比較表

| 観点 | 案A | 案B |
|---|---|---|
| 初見の分かりやすさ | ◎ 何をするサイトか即分かる | ○ 雰囲気は良いが学習サービス感は弱まる |
| 1タップ目の迷い | ○ CTA1＋副導線1 | ◎ CTA1のみ |
| モードを選びたい人 | ○ fold下ですぐ | △ 1スクロール必要 |
| はちゃん愛着の醸成 | △ 現状と同じ | ◎ 世界観の中心 |
| v4.1将来構想（季節/衣装/イベント）への拡張 | △ 作り直しが必要 | ◎ シーン領域がそのまま土台 |
| 「学習を邪魔しない」原則 | ◎ | ○（シーンが縦幅を取る） |
| 実装コスト | 小（1〜2日） | 中（3〜5日） |
| リスク | 低（既存の再配置） | 中（初見ユーザーに用途が伝わらない可能性） |

## 推奨

**10人ローンチは現状ホームのまま実施し、観察後に決める**のが最も安全。その上で選ぶなら:
- 観察で「何をすればいいか迷う」が出た場合 → **案A**（迷い解消が直接目的）
- 観察で「はちゃんへの好反応」が出た場合 → **案Aの構造に案Bのシーン領域を段階移植**（A→Bはアップグレードパスがある。B→Aは後退に見える）

つまり **A基盤→B要素を段階追加** が推奨ルート。
