# はちゃん（はすみ / Hasumi）公式キャラクター仕様

> 2026-07-16 創業者決定・正式仕様。キャラクター表現に関する判断はすべてこの文書が正。

## 0. 前提思想

SpellDashは「英単語学習アプリ」ではなく **『世界一継続される英単語サービス』** を目指す。
単純な学習効率より、**毎日開きたくなる理由・感情的な愛着・小さな成功体験・習慣化** を最優先する。

## 1. プロフィール

| 項目 | 内容 |
|---|---|
| 名前 | はすみ（Hasumi） |
| 愛称 | はちゃん |
| 役割 | 未来サポーターAI（先生ではなく**学習パートナー**） |
| 性格 | 優しくて落ち着いているけど、喜怒哀楽がとっても豊か |
| 好きなこと | おいしいもの、カフェ巡り、みんなの「できた！」を見ること |
| 夢 | みんなと一緒に、もっといろんな景色を見に行くこと |
| キャラクターカラー | #8B6A4F / #A67C5B / #EAD7C6 / #CDB4DB / **#8D70FF** / #6E60A9 |

**立ち位置**: ユーザーが主人公。はちゃんは一緒に頑張る伴走者。**はちゃん自身が主役になってはいけない。**
役割は「応援・共感・励まし・成長を一緒に喜ぶ」のみ。

**目指す感情**: ユーザーが「今日は疲れたけど、5分だけやろうかな。」と思える存在。勉強を強制する存在ではない。

## 2. 表現ルール（NG / OK）

### 絶対NG（この方向に進めない）

- 萌えゲーム化・アイドル化・過度な恋愛要素
- 依存を煽る表現・嫉妬表現・束縛表現
- 罪悪感を与えるセリフ

NG例: 「来てくれないと思った…」「寂しかった…」「会いたかった…」

### OK例

「おかえり！」「今日も少しだけ頑張ろう！」「前より成長してるよ！」

## 4. 実装フェーズ

- **Phase1（実装済み）**: ホーム一言・結果画面・ストリークのみ。ユーザー反応を見る
- **Phase2（反応を見てから検討）**: 季節衣装・スタンプ・シェア画像・イベント

実装の実体:
- `js/hasumi.js` — セリフ選択＋吹き出しUI（トーン規則をコードコメントにも明記）
- `assets/images/hasumi.png`（通常）/ `hasumi-happy.png`（お祝い） — キャラシートから切り出した丸アバター

## 5. App Store戦略（アプリ化時）

アイコンに「はちゃん」を採用予定。「勉強アプリ」ではなく **「一緒に頑張るパートナー」** という第一印象を作る。

## 6. キャッチコピー

- 第一候補: **「今日も、はちゃんと少しだけ。」**
- その他: 「あなたの英語学習の、いつもの隣に。」「気付いたら、英単語が好きになっている。」「一緒に未来へダッシュしよう。」

## 7. 新機能追加時の判断基準（必ず自問する）

1. 毎日開く理由になるか？
2. 学習体験を邪魔しないか？
3. **はちゃんが主役になっていないか？**
4. 習慣化につながるか？
5. 愛着が生まれるか？

## 8. スマホUI原則（キャラ表示にも適用）

- 主戦場はスマホ（通勤・通学・寝る前・SNS導線）。PC最適化よりスマホ体験優先
- **スクロールしない。1画面1目的。学習の邪魔をしない**
- Study画面に置いてよいもの: 残り時間・単語・入力欄・最小限の情報・**小さな**はちゃん
- UIは極限まで削る
