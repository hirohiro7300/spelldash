# SpellDash ロードマップ更新案（2026-07-18、v4.1 §12-4）

> 前提順序（v4.1指定）: 小規模ローンチ → ミスタイプ判定 → ホーム画面 → Challenge強化 → ストーリー最小実装 → 単語データ改善 → 学年別コース → 実用英語シリーズ → キャラクター課金。
> 各ステップに「開始条件」と「実装単位」を付けた。すべて本番反映は都度承認制。

## Phase 0: 小規模ローンチ（今すぐ・ユーザー側）
- 10人配布（社会人5/学生3/関心薄2）・7日間観察・FB記録（LAUNCH.mdキット使用）
- 並行: Vercel重複プロジェクト整理（確認→承認→削除）
- 開発側は観察待機。B優先度（観察で判明した初回体験の修正）だけ即応

## Phase 1: ミスタイプ判定（Clean Correct）
- **開始条件**: ローンチ配布が動き出したら並行着手可（学習データの信頼性はFB収集と独立して価値がある）
- 実装単位: ①SQL 1行（列追加・承認制）→ ②sync対応 → ③記録＋リザルト表示 → ④E2E 6件（SPEC_CLEAN_CORRECT.md）
- Dailyシェア絵文字の細分化（🟩/🟨の意味変更）は**別判断**（見た目が変わるため創業者承認）

## Phase 2: ホーム画面
- **開始条件**: 10人観察の初回所見が出てから（案A/Bの選択材料になる）
- 推奨ルート: 案A（学習導線・小コスト）を基盤に、はちゃん好反応が確認できたら案Bのシーン領域を段階移植（HOME_WIREFRAMES.md）
- 実装単位: おすすめCTAカード → 再配置 → （後続）シーン領域

## Phase 3: Challenge強化
- **開始条件**: Clean Correct導入後（スコアとClean分離が先。演出は判定の上に載せる）
- 順序（AUDIT_CHALLENGE.md §8、1つずつ検証）:
  1. 単語間250ms待ち撤廃（体感最大・コスト極小）
  2. プレイ中スコア＋コンボのカード内常設
  3. コンボ段階演出（数字スケール＋SFXピッチ）
  4. ベスト差分表示
  5. フィーバー（1〜4の反応を見てから）
- 制約: 入力視認性不変・重い演出禁止・reduced-motion対応

## Phase 4: ストーリー最小実装
- **開始条件**: はちゃんPhase2解禁条件（V4 §5: 自発的言及3件+等の2つ以上）
- 最小単位: 状態リアクション・表情差分（既存キャラシートから切り出し）・学習日数での一言変化。エピソード解放はその後

## Phase 5: 単語データ改善
- **開始条件**: 「訳が曖昧で答えられない」FBが実際に観察されたら前倒し可（それまでは設計のみ）
- 手順: データスキーマ拡張設計（品詞・代替正解・例文・出典）→ 曖昧訳の洗い出し（機械抽出: 訳がカタカナ1語などをvalidate-wordsで検出）→ 段階置換。外部教材の自動取得は禁止のまま、出典と著作権を先に確定

## Phase 6: 学年別コース
- **開始条件**: Phase 5のスキーマに「学年/CEFR」列が入ってから。教育課程準拠の分類ソース確定が先（UI先行禁止）
- 最初は「中学英語総復習」「大人の学び直し」の2コースから（既存カテゴリと親和的）

## Phase 7: 実用英語シリーズ「生きた英語」
- **開始条件**: Phase 5スキーマ（使用場面・丁寧さ・注意の併記構造）完成後
- 最初は「日常会話」「仕事で使う英語」の安全な2本から。スラングは情報設計が揃うまで出さない

## Phase 8: キャラクター課金
- **開始条件**: V4 §9一次ゲート（累計100人・継続20人・週3回10人・有料要望5件）→ まずユーザーインタビュー
- 対象は愛着・表現・世界観のみ（衣装・背景・演出・シェアデザイン）。Pay to Win禁止。通貨/ガチャ/ショップは引き続き禁止

## 横断ルール
- 新機能はv4 §12の4条件（課題・観察有無・既存改善で不可か・増える情報）を添えて提案
- 1PR=1検証単位。E2E必須。マージはバッチ1回=デプロイ1回
- 各Phaseの完了ごとにSTATUS.md更新
