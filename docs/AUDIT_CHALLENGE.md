# Challenge現状監査（2026-07-18、v4.1 §12-1）

> コード実測に基づく監査。対象: js/game.js / ui.js / sfx.js / stats.js / css/game.css。実装変更なし。

## 1. 現在のゲームループ（Challenge/Daily共通経路）

```
setNewWord（重み付き抽選 or Dailyは固定セット順）
 → 日本語訳を表示・入力欄フォーカス
 → 1文字ずつ照合（desktop: keydown / mobile: inputイベント差分）
 → 全文字一致 → completeWord()
     score++ → SFX → 記録（recordCorrect等）→ XP計算 → メッセージ表示
     → setTimeout(setNewWord, 250) ★次の単語まで一律250ms待ち
 → 60秒経過 → endChallenge() → リザルトパネル
```

- タイマーはsetIntervalで残秒表示（big-timer、残10秒で赤＋pulse）
- Enter1回目=答え表示（recallFail・combo=0）、答え表示後の完答も score++（🟨扱い）

## 2. 正解処理の実測

| 項目 | 現状 |
|---|---|
| スコア | **1語=1点の語数カウント**（clean/typo/答え見た後も同じ1点） |
| clean判定 | `!hasMissedCurrentWord && !isRevealed` — completeWord内で判定済み |
| 学習記録 | `recordCorrect(id, isClean)`: **SRS/mastered前進はcleanのみ**（stats.js）。typo正解はcleanCorrectStreak=0＋翌日復習 |
| XP | 自力正解=10+(clean5)+min(combo,10)。答え見た後=5 |
| コンボ | **cleanのみ**combo++。typo/reveal/missでリセット。表示は2以上で「🔥 N COMBO」、5以上で色変化のみ |

## 3. ミスタイプ処理の実測

- typingMissCount++・combo=0・sfxMiss・「Miss!」表示・recordTypingMiss（苦手判定には不使用）
- 入力欄はそのまま（正しい文字を打ち直すまで進まない）。モバイルは受理済みプレフィックスへ巻き戻し

## 4. 演出の現状

- あるもの: スコア数字パルス（stat-pulse）・コンボテキスト・正解/ソフト正解/ミス/リビールのSFX 4種・big-timer残10秒赤パルス・リザルトパネル（ベスト更新バッジ🏆・はちゃん一言）
- ないもの: コンボの成長演出（5/10/20での変化はCSS色1段のみ）・フィーバー・ベスト差分のリアルタイム表示・終了間際の音変化・連続正解での効果音ピッチ上昇（sfxCorrect(combo)はコンボを受け取るが変化は小さい）

## 5. スマホ上の問題（実測・スクショ確認済み）

1. **✅解消済み**: フォーカスモードでプレイ中は単語+入力+タイマーのみ
2. スコアが下の統計カードにあり、**プレイ中の視界に入らない**（打鍵中に見えるのは残り時間のみ）
3. コンボ表示が入力欄の下の小さいテキストで、キーボードに隠れる位置になりうる
4. メッセージ行（"Good! +18 XP"）が毎語書き変わるが、視線は単語にあるためほぼ読まれない

## 6. テンポの計測値（最重要の発見）

- **単語間に一律250msの固定待ち**（`setTimeout(setNewWord, 250)`）
- 60秒で30語完答した場合: 30×250ms = **7.5秒（ランの12.5%）が待ち時間**
- 上級者ほど待ち比率が上がる＝「速く打つほど待たされる」構造
- 判定自体は同期的で速い（keydown即時）。ボトルネックはこの固定待ちのみ

## 7. ツムツム的快感構造から応用できる要素（SpellDash向けに翻訳）

| ツムツムの構造 | SpellDash翻訳 | コスト |
|---|---|---|
| 消した瞬間の即時応答 | 単語間250ms待ちの短縮/撤廃（正解SFXは鳴らしつつ次の単語を即表示） | 極小 |
| スコアが常に動いて見える | プレイ中のゲームカード内に現在スコア+コンボを常設表示（big-timerの対角） | 小 |
| コンボ数の成長感 | コンボ数字のサイズ/色が5,10,20で段階変化＋SFXピッチ上昇 | 小 |
| フィーバータイム | 10コンボで数秒間スコア倍率＋カード縁が光る（入力欄は不変） | 中 |
| 終了間際の追い込み | 残10秒: 既存の赤タイマーに加えBGM/SFXテンポ変化 | 小 |
| 結果画面の「もう1回」即導線 | ✅実装済み（リザルトパネル）＋ベスト差分「あと12点でベスト」追加 | 小 |

## 8. 改善余地の優先順位案（最小変更で快感が増える順）

1. **単語間250ms待ちの撤廃（Challengeのみ）** — 実装1行級・体感最大。正解演出は非同期で重ねる
2. **プレイ中スコア＋コンボのカード内常設**（big-timer左上の対角=右上に小型表示）
3. **コンボの段階演出**（数字スケール＋SFXピッチ。CSSとsfx既存関数の拡張のみ）
4. ベスト差分表示（リザルト＋残20秒時に「ベストまであとN」）
5. フィーバー（10コンボ×5秒×スコア1.5倍等）— 上4つの反応を見てから

※v4.1の制約遵守: いずれも入力欄・単語の視認性を変えない。アニメはCSS transform/opacityのみ（reduced-motion対応）。一括実装せず、1→2→3の順に1つずつ検証。
