# SpellDash v4 上流設計決定（2026-07-18）

> 現在の最上位方針。PMF.md / CHARACTER.md / LAUNCH.md はこの決定に従属する。

## 0. フェーズ再定義

**「小規模公開・観察中心のPMF探索」**へ移行。主要機能は十分に揃っているため、新機能追加より実ユーザーからの情報の方が価値が高い。最優先: ①実際に人へ配る ②初回利用を観察 ③翌日以降に戻るか確認 ④迷った箇所を修正。

## 1. 小規模ローンチ設計

- **10人**限定（一人ひとりの行動を追える人数）
- 内訳: 英語学び直し社会人5 / 学生・資格学習者3 / 英語に関心が薄い人2（ゲームとして自然に遊べるかを見る）
- チャネル優先順: 直接連絡できる知人 → X個人アカウント → 家族・友人経由 → 広い告知。広告なし
- 観察期間: 初回プレイから**7日間**。観察項目: 初回に選んだモード / 説明なしで遊べたか / 迷った箇所 / 翌日以降の自発的復帰 / 覚えていた機能 / はちゃんへの反応

## 2. KPI計測の再開条件（いずれか）

- 累計登録・識別可能ユーザー30人 / 累計プレイユーザー50人 / 定性FB10件
- 満たすまでD1/D7を意思決定の主根拠にしない。**収集は継続、評価はしない**

## 3. メインターゲット

**社会人の英語学び直し層**（開発者自身が当事者・「仕事終わりに少しだけ」と一致・課金可能性）。中高生は排除しないが、コピー・オンボーディング・SNS発信は社会人に寄せる。利用場面: 通勤・休憩・帰宅後・寝る前。

## 4. コピー戦略（初見向けと継続向けを分ける）

- ヒーロー（初見向け）: **「思い出して打つから、英単語が残る。」**＋補助「日本語を見て、英単語をタイピング。1日少しずつ、英語とタイピングを同時に鍛える。」 ✅実装済み
- 「今日も、はちゃんと少しだけ。」は継続ユーザー向け: 結果画面 ✅ / SNS投稿（Dailyシェア文）✅ / ログイン後ホーム・PWA起動時（今後）

## 5. はちゃんPhase2解禁条件（2つ以上で再検討）

自発的言及3件+ / セリフが継続理由に挙がる / シェア画像のキャラ要素に好意的反応 / 「邪魔」の声がほぼない / 継続ユーザー10人+。
解禁後の最初は**表情差分・状態リアクション・シェア画像差分**（衣装ではない）。キャラコンテンツの大量生産は禁止。

## 6. 復帰導線

通知なしで戻るかをまず確認（通知による復帰とプロダクト魅力による復帰を分離）。解禁条件: 累計30人+ ＆ D1計測可能 ＆「通知が欲しい」3件+。実装時は**PWA Pushのみ**候補、メールは低優先。

## 7. Battle

削除しない・当面改善しない。役割=**学習成果を試す場所**（副次モード）。主要導線に置きすぎない。利用率が高ければ改善 / 初回だけなら導線を下げる / Battleだけ遊ばれるなら学習接続を再確認。少人数フェーズでは利用率だけで削除判断しない。

## 8. アプリ化

現時点でしない。検討条件: 累計プレイ100人 / 週3日利用20人 / スマホ率70%+ / インストール・アプリ希望10件+ / 専用KBが体験を大きく改善すると判断できる。PWAで解決できない明確な課題が出た場合のみ。専用キーボード構想はdocs保持・実装禁止。

## 9. 収益化

課金実装禁止を維持。旧ゲート（DAU100/D1 20%/Daily30%）は不採用。一次ゲート: 累計100人 / 継続20人 / 週3回利用10人 / 有料要望5件+。満たしたら課金実装ではなく**有料プランのユーザーインタビュー**を先に実施。¥580/月は仮説として保持（正式決定しない）。

## 10. Vercel重複プロジェクト

早急に整理。削除前確認: spelldash.netの接続先 / GitHub連携先 / 環境変数 / デプロイ履歴 / Production Domain / mainのデプロイ先。正規を特定後、不要側のみ削除。**作業は創業者承認後**。

## 11. 実装優先順位

- **A**: ①承認待ちコミット反映 ✅（PR#33）②Vercel整理 ③10人ローンチ実行 ④初回観察 ⑤FB記録
- **B**: 観察で判明した初回体験・スマホ操作・説明不足の修正
- **C**: 軽微なビジュアル改善（新規イラスト大量追加なし）

## 12. 開発停止ルール

FBなしに新機能を連続追加しない。新機能提案には必ず: どのユーザー課題か / その課題は観察されたか / 既存機能の改善で解決できないか / PMF探索に増える情報は何か。「面白そう」「競合にある」「将来必要」だけでは実装しない。

## 13. 直近の成功条件

売上でもDAUでもなく**10人に触ってもらう**。うち: 3人が翌日再訪 / 1人が説明なし3日+利用 / 1人が紹介 / 具体的改善点5件 — いずれかで次の仮説検証へ。何も起きなくても失敗ではなく、初回体験・価値訴求・対象ユーザーのズレを分析する。

## 14. 判断軸（全判断はこの順）

1. また明日も開きたくなるか 2. 英単語を実際に思い出せるようになるか 3. スマホで負担なく遊べるか 4. はちゃんが体験を支えているか 5. 実装・運用を小さく保てるか

ビジョン『世界一継続される英単語サービス』は不変。ただし今は世界ではなく**最初の10人**を見る。
